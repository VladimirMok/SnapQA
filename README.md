# SnapQA
Ускорьте создание баг‑репортов с SnapQA: один клик — и скриншот с контекстной информацией (время, окно, процесс). Версии под Windows, Linux или универсальная сборка.

# Скриншоты с информацией об активном окне


Программа для создания скриншотов с автоматической маркировкой: время, название активного окна и процесса. Доступны три версии для разных ОС.


## Версии

1. **`screenshot_w.py`** — оптимизирована для **Windows**. Использует Windows API для получения информации об активном окне и процессе.
2. **`screenshot_l.py`** — адаптирована для **Linux**. Использует `xdotool` для получения информации о текущем окне.
3. **`screenshot.py`** — **универсальная** версия. Автоматически определяет ОС и использует соответствующий метод.


## Функционал


* Делает скриншот всего экрана.
* Добавляет текстовую метку с:
  * текущим временем;
  * названием активного окна;
  * именем процесса‑владельца окна.
* Сохраняет файл в указанную папку с осмысленным именем.
* Формат имени файла: `ЧЧММ_n_ДДММ_Процесс_Окно.png` (например, `0748_0_0503_WINWORD_ПАССЫ.png`).
* Горячие клавиши:
  * **Создание скриншота:** `Ctrl+Alt+Пробел`.
  * **Выход из программы:** `ESC`.

Использование

    Клонируйте репозиторий:
    bash

    git clone <URL-вашего-репозитория>
    cd <название-папки-проекта>

    Запустите нужную версию скрипта:

        Windows:
        bash

        python screenshot_w.py

        Linux:
        bash

        python screenshot_l.py

        Универсальная версия (автоопределение ОС):
        bash

        python screenshot.py

    При запуске программа попросит указать папку для сохранения скриншотов:

        Нажмите Enter, чтобы использовать папку по умолчанию (screenshots).

        Введите название папки (например, my_shots), чтобы сохранить в screenshots/my_shots/.

    Делайте скриншоты:

        Нажмите Ctrl+Alt+Пробел в любой момент.

        Скриншот с меткой будет сохранён в выбранной папке.

    Выйдите из программы:

        Нажмите клавишу ESC.

Формат имени файла

ЧЧММ_n_ДДММ_Процесс_Окно.png

    ЧЧММ — часы и минуты создания скриншота.

    n — порядковый номер скриншота в течение одной минуты (предотвращает дубликаты).

    ДДММ — день и месяц.

    Процесс — имя исполняемого файла процесса (без расширения .exe на Windows).

    Окно — заголовок окна (обрезан до 15 символов для краткости).

Пример: 0748_0_0503_WINWORD_ПАССЫ.png.

Примечания

    Права доступа. На Linux может потребоваться запуск скрипта с правами, позволяющими перехватывать глобальные горячие клавиши.

    Шрифты. Скрипт пытается использовать arial.ttf. Если его нет, используется шрифт по умолчанию.

    Уникальность. Порядковый номер (n) гарантирует, что файлы не перезапишутся, даже если сделать несколько скриншотов за минуту.
